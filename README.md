


# Using the project

Each header file is a self contained full application file, commented out so the project will still build.
Throughout the presentation I would simply copy the contents of one of the header files into the application
file and rebuild the project. External classes that came in later can be left alone and won't break the project.
Some files contain more code that I purposely left commented out during the presentation and would uncomment
when applicable. There are not many instances of this and most should just run.


###### I included the dependencies I used, so if you're using windows and the Visual Studio / VSCode C++ compiler then I think it might work. Apologies for my newbie C++ skills 🙃.
